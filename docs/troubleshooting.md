# Troubleshooting Guide

This document describes how to debug, resolve, and prevent common operational issues when developing and deploying the Customer Churn MLOps platform.

## 1. Docker Context Bloat (Slow Builds)
* **Issue**: Running `make build-image` prints:
  `Sending build context to Docker daemon 2.03GB...` and hangs or takes minutes to build.
* **Root Cause**: The `.venv/` virtualenv directory, git logs, raw dataset caches, or DVC caches are being sent to the Docker daemon because they are not ignored.
* **Resolution**: Ensure a `.dockerignore` file exists in the repository root and excludes the `.venv/`, `data/`, `.dvc/`, and all cache directories (such as `.pytest_cache/` and `.mypy_cache/`). Check [.dockerignore](file:///home/ubuntu/mlops/mlops-customer-churn-model/.dockerignore) for the full list of excluded paths.

## 2. MLflow Database Connection Failures
* **Issue**: The `mlflow` container logs show:
  `sqlalchemy.exc.OperationalError: (psycopg2.OperationalError) could not connect to server: Connection refused`
* **Root Cause**: The MLflow tracking server started before the PostgreSQL database container was fully initialized and ready to accept connections.
* **Resolution**: The `mlflow` service in `docker-compose.yml` must use a healthy dependency rule pointing to the `postgres` healthcheck:
  ```yaml
  depends_on:
    postgres:
      condition: service_healthy
  ```
  Ensure the `postgres` container has a working `healthcheck` defined (using `pg_isready`).

## 3. MinIO S3 Credentials Errors in DVC
* **Issue**: DVC uploads (`dvc push`) or pipeline stages fail with AWS S3 signature or credential errors.
* **Root Cause**: Local AWS environment variables (like those generated by AWS SSO or AWS CLI tools) interfere with DVC S3 MinIO backend connection.
* **Resolution**: Override or clear local environment credentials by setting the explicit MinIO credential parameters before running pipelines:
  ```bash
  export AWS_ACCESS_KEY_ID=minioadmin
  export AWS_SECRET_ACCESS_KEY=minioadminpassword
  export MLFLOW_S3_ENDPOINT_URL=http://localhost:9000
  export MLFLOW_S3_IGNORE_TLS=true
  ```

## 4. vCluster API Server Unreachable
* **Issue**: Commands like `kubectl get namespaces` return connection refused when targetting the vCluster.
* **Root Cause**: The local vCluster instance is paused or the port forwarding tunnel is closed.
* **Resolution**:
  1. Check the status of your vCluster:
     ```bash
     vcluster list
     ```
  2. Resume the vCluster if it is paused:
     ```bash
     vcluster resume my-cluster
     ```
  3. Re-establish cluster context and port-forwarding:
     ```bash
     vcluster connect my-cluster
     ```

## 5. Drift Detector Fails to Spot Real Drift
* **Issue**: Shifted categorical distributions in testing or production are not detected as data drift by Evidently AI.
* **Root Cause**: Evidently's default `DatasetDriftMetric` marks dataset drift as True only when more than **50%** of total features exhibit statistical drift. If only a few columns drift, the dataset is marked as non-drifted.
* **Resolution**:
  - Shift a larger number of features (e.g. categorical mappings alongside numerical values) to test drift triggering.
  - Or override the drift ratio threshold in Evidently configuration or in the retraining script parameter (`--threshold 0.3`).
