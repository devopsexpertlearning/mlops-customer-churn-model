# Artifacts Directory

## 1. Purpose
This directory stores intermediate pipeline output files (such as validation reports, metric files, data profiling schemas, and evaluation curves) produced during local execution.

## 2. Architecture and Workflow
Pipeline scripts write status details or metrics to this folder. These files are versioned by DVC and pushed to S3 object storage in production, keeping large and volatile metric structures out of Git.

```text
 [Validation Module] ──> [artifacts/validation_report.json] ──> Tracked by DVC
```

## 3. Files Contained Inside
* [validation_report.json](file:///home/ubuntu/mlops/mlops-customer-churn-model/artifacts/validation_report.json): Generated by `validate.py` during validation checks, mapping schema compliance, value ranges, and target class distributions. **This file is git-ignored.**
* `app.json.log`: Structured JSON logs (configured by `configs/logging_config.json`). **This file is git-ignored.**

## 4. Interaction with Other Modules
* **Validation Module** ([validate.py](file:///home/ubuntu/mlops/mlops-customer-churn-model/src/customer_churn/validate.py)): Outputs validation status to this directory.
* **Evaluation Module** ([evaluate.py](file:///home/ubuntu/mlops/mlops-customer-churn-model/src/customer_churn/evaluate.py)): Will output `metrics.json` here in later phases.

## 5. Usage Instructions
To generate or update artifacts:
```bash
dvc repro validate
```

## 6. Examples
### Reading the validation report
```python
import json
with open("artifacts/validation_report.json") as f:
    report = json.load(f)
print(f"Validation Status: {report['validation_passed']}")
```
